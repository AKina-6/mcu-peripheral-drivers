# 基本定时器与周期任务学习笔记

## 定时器结构

```text
输入时钟 → PSC → CNT → ARR → 更新事件 → 中断
```

## 更新频率

```text
f_update = f_timer / ((PSC + 1)(ARR + 1))
```

## 一个硬件 Tick 驱动多个软件周期

```text
1 ms Tick
├── 500 Tick  → LED
├── 1000 Tick → Sensor
└── 2000 Tick → Telemetry
```

这种方式节省硬件定时器，并为后续环境监测项目提供基础调度框架。

## 中断设计

ISR 中建议只做计数、置标志、记录时间或通知任务。传感器读取、OLED 刷屏、串口发送等操作放到主循环或 RTOS 任务。

## 事件计数与布尔标志

布尔标志可能丢失重复事件；计数器可以记录积压次数，但需要考虑上限和积压处理策略。

## HAL_Delay 的局限

`HAL_Delay()` 会阻塞当前执行流程。周期事件方法可以让主循环持续响应其他模块。
